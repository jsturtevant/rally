## Work Summary

Mal drafted the complete Product Requirements Document (`docs/PRD.md`) establishing the foundational architecture and command structure for Rally CLI:

### Core Decisions Made

1. **Three-file state model** — File-based, zero dependencies, human-readable configuration
   - `~/.rally/config.json` — Team setup
   - `~/.rally/projects.json` — Project registrations
   - `~/.rally/active.json` — Active worktrees & assignments

2. **Symlink + exclude as the foundation** — Portable team state mechanism
   - `rally onboard` creates symlinks to team state
   - Writes to `.git/info/exclude` to prevent accidental commits
   - Enables "don't commit Squad state" workflows

3. **Worktrees inside the repo** — Isolated branches and working directories
   - Created at `.worktrees/rally-<N>/`
   - Branch naming: `rally/<N>-<slug>`
   - Automated creation, management, and teardown

4. **Module-per-command structure** — Clean, maintainable codebase
   - `lib/` directory with separate modules for each command
   - Shared utilities for state management, git operations, validation
