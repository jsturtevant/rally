## Task 2: Restructure dispatch as subcommands
**Status:** Completed

### Work Done
- Replaced positional arg + flag syntax with explicit subcommands:
  - Old: `rally dispatch <issue-number>` (implicit issue)
  - Old: `rally dispatch --pr <pr-number>` (flag-based PR)
  - New: `rally dispatch issue <number>` (explicit subcommand)
  - New: `rally dispatch pr <number>` (explicit subcommand)
- Both subcommands accept optional `--repo <owner/repo>` to specify target repo
- Inference logic: cwd → `projects.yaml` (single project) → error
- Updated all affected sections in `docs/PRD.md`: §3.3, §3.4, §4.2, Appendix A

### Rationale
- Self-documenting syntax
- Symmetric structure (no special-cased flag for one mode)
- `--repo` uses GitHub convention (owner/repo) instead of local paths
