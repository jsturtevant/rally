## Technical Details

**Problem:** Fixed `delay(50)` calls in multi-step UI test interactions caused timing-dependent failures on slower systems.

**Solution:** Implemented `waitForFrame(expected, options)` helper that:
- Polls the UI component's frame content at 50ms intervals
- Waits for expected text/content to appear
- Supports configurable timeout (default 2000ms) and poll interval
- Throws descriptive error if content not found within timeout
